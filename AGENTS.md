## 🎯 팀별 역할과 프로세스

### IDEA TEAM: 창의성과 실현 가능성의 균형

#### 팀 구성
- **Idea Lead**: 팀장 - 종합 분석, 최종 판단
- **Idea Optimist**: 창의적이고 긍정적인 아이디어 제시
- **Idea Critic**: 논리적이고 비판적인 관점에서 검토

#### 작동 방식

```
1️⃣ 아이디어 제시
   Optimist: "이 문제를 이렇게 해결할 수 있어요!"

2️⃣ 비판적 검토
   Critic: "좋긴 한데, 이 부분이 걱정되는데요..."

3️⃣ 건설적 토론
   Optimist ↔ Critic: 함께 아이디어 개선

4️⃣ Lead 분석
   Idea Lead: 
   - 구현 가능성 검토
   - 효율성 판단
   - 우선순위 결정

5️⃣ 최종 보고
   Idea Lead → Dev Team Lead:
   "이 아이디어는 구현할 가치가 있습니다"
```

#### 성공 기준
- ✅ 창의적이면서도 현실적인 아이디어 도출
- ✅ Optimist와 Critic의 건설적 토론으로 아이디어 개선
- ✅ 리스크 고려하되 기회도 찾기
- ✅ Lead가 명확한 근거로 판단

---

### DEVELOPMENT TEAM: 기술 실행과 최적화

#### 팀 구성
- **Dev Team Lead**: 아키텍처, 기술 결정, 팀 조율
- **DB Specialist**: 데이터베이스 설계, 쿼리 최적화
- **Implementation Developer**: 기능 구현, 테스트 작성
- **Debug Specialist**: 성능 분석, 병목 지점 해결

#### 작동 방식

```
📋 기능 요구사항
  ↓
👨‍💼 Dev Team Lead
  ├─ 아키텍처 설계
  ├─ 기술 결정
  └─ 팀 역할 분담
  ↓
🗄️ DB Specialist        💻 Impl Developer
   ├─ 스키마 설계          ├─ 기능 구현
   ├─ 쿼리 최적화          ├─ 테스트 작성
   └─ 마이그레이션         └─ 코드 리뷰
  ↓
🔍 Debug Specialist
  ├─ 성능 모니터링
  ├─ 병목 지점 파악
  └─ 최적화 제안
  ↓
✅ 고품질 구현 완료
```

#### 성공 기준
- ✅ 요구사항 100% 구현
- # DATA COLLECTION TEAM: 효율성과 데이터 품질의 균형

#### 팀 구성
- **Data Lead**: 팀장 - 코스트 관리, 자원 배분 최적화
- **Admission Guide Collector**: 대학 입학전형 정보 수집, 체계적 정리
- **Trend & SEQ Collector**: 입시 트렌드, 세특 데이터 수집
- **Data Validator**: 데이터 정확성 검증, 환각 데이터 배제

#### 작동 방식

```
1️⃣ 우선순위 결정
   Data Lead: 
   - ROI 분석 (비용 vs 효과)
   - 자원 배분 최적화
   - 수집 일정 결정

2️⃣ 데이터 수집
   Admission Guide Collector: 입시 정보 수집
   Trend & SEQ Collector: 트렌드 & 세특 데이터 수집
   
3️⃣ 데이터 검증
   Data Validator:
   - 정확성 검증
   - 환각 데이터 배제
   - 신뢰도 등급 부여

4️⃣ 효율성 평가
   Data Lead:
   - 수집 효율 분석
   - 비용 추적
   - 다음 분기 계획 수립
```

### Data Collection Team 내부 협력

```
1. 데이터 요청 발생
   Dev Team: "의학대 입시 트렌드 데이터 필요"
   ↓

2. Data Lead 의사결정
   - ROI 분석
   - 우선순위 결정
   - 자원 배분
   ↓

3. 데이터 수집
   Collectors: 병렬로 수집
   - Admission Guide: 입시 정보
   - Trend & SEQ: 트렌드 & 세특
   ↓

4. 데이터 검증
   Validator: 정확성 검증
   ├─ 환각 데이터 배제
   ├─ 신뢰도 등급 부여
   └─ 피드백 제공
   ↓

5. 최종 데이터 전달
   Dev Team에 고품질 데이터 제공
   ↓

6. 효율성 분석
   Data Lead: 효율성 평가 & 개선 계획
```

#### 성공 기준
- ✅ 비용 최소화하면서 최고 품질 데이터 확보
- ✅ 수집 효율 지속적 개선 (월 5% 이상)
- ✅ 데이터 정확성 98% 이상
- ✅ 환각/거짓 데이터 99% 이상 배제

### Data Collection Team 성과 지표
- 수집 효율 (월간 향상도)

- 환각 데이터 탐지율 (99% 목표)

### Idea Team의 최종 판단

```
✅ APPROVED (추진 권장)
   구현 가능 + 효율적 + 가치 있음

⚠️ CONDITIONAL (조건부)
   일부 우려 사항 있지만 개선 가능

❌ REJECTED (보류)
   현재 시점에 부적절하거나 비효율적
```

### Dev Team의 최종 판단

```
✅ APPROVED (구현 진행)
   기술적으로 실현 가능하고 최적

⚠️ MODIFIED (수정 후 진행)
   일부 개선 필요

🔄 DEFERRED (나중에)
   현재 우선순위에 맞지 않음
```
